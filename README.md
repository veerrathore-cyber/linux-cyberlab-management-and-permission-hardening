# Linux CyberLab Management and Permission Hardening

## Project Overview

This project demonstrates the practical application of Linux file system management and security principles by creating a structured cybersecurity lab environment.

## Objectives

- Create a cybersecurity directory hierarchy.
- Organize tools and reports efficiently.
- Apply Linux permission hardening techniques.
- Document real-world cybersecurity incidents.

## Tools Used

- Kali Linux
- mkdir
- touch
- chmod
- ls

## Tasks Performed

### Task 1: Directory Hierarchy Creation
- Created a CyberLab directory structure.
- Used brace expansion for efficient folder creation.
- Generated case study files.

### Task 2: Permission Hardening
- Applied chmod 700 to sensitive directories.
- Verified access permissions.

### Task 3: Cybersecurity Case Studies
- MGM Resorts Cyberattack
- Change Healthcare Ransomware Attack
